# Project Brief for Antigravity

## **Project Name**

**Operating Systems Museum**

## **Vision**

Create a premium, museum-quality web application called **Operating Systems Museum**.

This is **not** just a website about operating systems. It should feel like an interactive digital museum where users can explore the history of operating systems through beautiful storytelling, immersive visuals, interactive timelines, technical information, comparisons, and browser-based emulation where legally possible.

The overall experience should feel closer to visiting a modern technology museum than browsing a documentation website.

The design should be elegant, premium, highly polished, minimal, and modern.

---

# **Languages**

The entire website must support:

- 🇺🇸 English
- 🇹🇷 Turkish

Provide a language switcher in the navigation.

Every page, title, description, button, and content should be prepared for localization.

Do not hardcode text directly inside components.

# **Launch Experience**

Each operating system should have a “Launch Experience” button.

Important:

Do NOT bundle or distribute copyrighted operating system images.

Instead, design the system to support:

User-provided disk images

Future emulator integration

Placeholder interface for unsupported systems

For open-source operating systems such as:

ReactOS

Haiku

FreeDOS

allow direct browser-based execution where legally appropriate.

For proprietary systems such as Windows, display a clean placeholder explaining that users can launch their own legally obtained system image when emulator support is enabled.

Design this architecture from the beginning so future emulator integration is easy.
